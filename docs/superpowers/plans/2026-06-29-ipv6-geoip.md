# IPv6 GeoIP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans / test-driven-development. Steps use checkbox (`- [ ]`).

**Goal:** Make `GeoIPDatabase.country(for:)` resolve IPv6 addresses to ISO country codes, and have the converter pack DB-IP's IPv6 ranges into `geoip.dat`, so the map/KPIs/flags stop under-counting IPv6 destinations.

**Architecture:** Binary format v2 appends an IPv6 range section after the unchanged IPv4 table (backward/forward compatible). Lookup dispatches on `IPAddress` family: IPv4 keeps its `UInt32` binary search; IPv6 binary-searches a `(UInt64,UInt64)` lexicographic range table. Everything downstream funnels through `country(for:)` and needs no change.

**Tech Stack:** Swift 6, Swift Testing, SPM module `MatrixNetGeoIP`, `Tools/GeoIPConvert` swift script, DB-IP Country Lite CSV.

**Interfaces:**

- Produces: unchanged public surface — `init?(data:)`, `country(for:) -> String?`, `isEmpty`, `static flag(for:)`. New internal `V6Range` + `init(v4Ranges:v6Ranges:)`.

- [ ] **Step 1: failing tests** — add to `GeoIPDatabaseTests.swift`:
  - `ipv6Lookup`: build a DB with a synthetic IPv6 range and assert a contained address resolves, an outside one is nil.
  - `ipv6BinaryFormatRoundTrip`: hand-assemble v2 bytes (v4count + 1 v4 record + v6count + 1 v6 record), `#require(GeoIPDatabase(data:))`, assert both an IPv4 and an IPv6 address resolve.
  - `legacyV4OnlyFileStillLoads`: v1 bytes (no v6 tail) → IPv4 resolves, an IPv6 address returns nil, `isEmpty == false`.
  - `ipv6LexicographicBoundaries`: range spanning a `high` boundary; assert endpoints inclusive and an address just past `end` is nil.
  - `ipv6PlaceholderIsNil`: v6 range with country "ZZ" → nil.
  - `emptyOnlyWhenBothTablesEmpty`: DB with empty v4 + empty v6 → `isEmpty == true`; v4-only → false.
  - `rejectsTruncatedV6Section`: v4count=0 then v6count=1 but no v6 record bytes → `init?` returns nil.
- [ ] **Step 2: run, verify FAIL** — `swift test --filter GeoIPDatabase` → compile error / failures (no IPv6 support yet).
- [ ] **Step 3: implement** in `GeoIPDatabase.swift`:
  - Add `struct V6Range { let startHigh, startLow, endHigh, endLow: UInt64; let country: String }`.
  - Store `private let v6Ranges: [V6Range]` alongside `ranges`.
  - `init(v4Ranges:v6Ranges:)` (rename existing `init(ranges:)` callers via a defaulted `v6Ranges: [] `).
  - `init?(data:)`: after reading the IPv4 table at `offset = 4 + count*10`, if `bytes.count >= offset + 4` read `v6count` (UInt32 BE) and `v6count` × 34-byte records (`start16`, `end16`, `cc2`), with a length guard `bytes.count >= offset + 4 + v6count*34` (else return nil). Pack 16 bytes → (high,low) big-endian.
  - `isEmpty`: `ranges.isEmpty && v6Ranges.isEmpty`.
  - `country(for:)`: switch on `address.unmappedIPv4` — `.v4` existing path; `.v6(high,low)` → `lookupV6(high:low:)` doing lexicographic binary search, placeholder-filtered.
  - Helper `private func lessOrEqual(_ aHigh:UInt64,_ aLow:UInt64,_ bHigh:UInt64,_ bLow:UInt64) -> Bool` or inline tuple compare `(aHigh,aLow) <= (bHigh,bLow)`.
